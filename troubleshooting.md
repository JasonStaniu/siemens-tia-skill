# TIA Portal 常见问题排查

本文件用于编译报错、下载失败、PLCSIM、HMI、通信问题的快速定位。

## 总排查顺序

```text
确认现象 → 拿到报错原文 → 分类 → 查最短链路 → 验证一个改动 → 再继续
```

优先要的信息：

- TIA 版本。
- CPU 型号和固件版本。
- 报错原文或截图关键信息。
- 是真实 PLC 还是 PLCSIM。
- 最近改过哪些块、硬件、网络或 HMI 变量。

## 编译报错

### 变量未定义

常见原因：

- 变量没有声明。
- 局部/接口变量缺少 `#`。
- 变量名拼写不一致。
- DB 名称或 UDT 名称错误。
- 引号、大小写或特殊字符不一致。

排查：

```text
打开报错位置 → 看变量声明区 → 检查 # 前缀 → 查交叉引用 → 检查 DB/UDT 名称
```

典型修复：

```scl
#Out_Run := #In_Start AND NOT #In_Stop;
```

### 数据类型不匹配

常见原因：

- `Int` 与 `Real` 直接混用。
- `Word` 当 `Bool` 用。
- `Time` 与数值直接运算。
- UDT 字段类型不一致。
- HMI 变量类型和 PLC 类型不一致。

排查：

```text
看等号左侧目标类型 → 看右侧表达式类型 → 必要时显式转换
```

常见转换：

```scl
#Temp_Real := INT_TO_REAL(#In_Int);
#Temp_Int := REAL_TO_INT(#In_Real);
```

### SCL 语法错误

常见原因：

- 漏分号。
- `IF` 没有 `END_IF`。
- `CASE` 没有 `END_CASE`。
- `FOR` 没有 `END_FOR`。
- 注释没有正确闭合。
- 数组下标写法错误。

排查：

```text
从第一条报错向上看 → 检查最近的 IF/CASE/FOR → 不要只看红线所在行
```

### FB 输出不是最新值

常见原因：

- 多实例赋值后没有调用。
- 在调用前读取输出。
- 上级 OB/FB 根本没有调用目标块。

正确写法：

```scl
#inst.Param := #Value;
#inst();
#Latest := #inst.Output;
```

### 中文乱码或导入异常

常见原因：

- `.scl` 不是 UTF-8 BOM。
- 从网页/聊天复制导致隐藏字符。
- 中文标点混入 SCL 语法位置。

修复：

```bash
python3 << 'PY'
from pathlib import Path
p = Path('example.scl')
text = p.read_text(encoding='utf-8-sig')
p.write_text(text, encoding='utf-8-sig')
PY
```

检查：

```bash
xxd -l 3 example.scl
# ef bb bf
```

## 下载失败

### 找不到设备

常见原因：

- PG/PC 接口选错。
- 电脑 IP 与 PLC 不同网段。
- 网线、交换机、虚拟网卡问题。
- 防火墙影响搜索。
- PLCSIM 未启动。

排查：

```text
设置 PG/PC 接口 → 可访问设备 → 检查本机 IP → 检查 PLC IP → 关闭干扰虚拟网卡/防火墙后重试
```

### 设备不匹配

常见原因：

- CPU 型号不一致。
- 固件版本不一致。
- 订货号不同。
- 项目硬件组态与实际机架/模块不一致。

排查：

```text
在线访问 → 识别设备 → 对比订货号/固件 → 必要时更换项目 CPU 或升级固件
```

### PROFINET 设备名不一致

表现：

- PLC 能找到，但远程 IO 或驱动不可达。
- 硬件诊断显示设备名称错误。

排查：

```text
在线访问 → 分配设备名称 → 下载设备名 → 重新下载硬件组态
```

### 下载会覆盖现场数据

风险点：

- DB 初始值覆盖实际累计量、配方、校准参数。
- CPU 停机导致设备停止。
- 输出变化导致现场动作。

处理原则：

```text
先备份 → 确认停机/安全 → 明确下载范围 → 获得授权 → 再执行
```

## PLCSIM 排查

### PLCSIM 无法下载

常见原因：

- PLCSIM 未启动或未创建实例。
- CPU 类型不支持当前 PLCSIM 模式。
- 项目硬件和仿真 CPU 不一致。
- PG/PC 接口不是 PLCSIM。

排查：

```text
启动 PLCSIM → 新建匹配 CPU → TIA 选择下载到仿真设备 → 搜索可访问设备 → 下载硬件和软件
```

### CPU 不 RUN

常见原因：

- 下载后仍 STOP。
- 诊断缓冲区有启动错误。
- OB 缺失或程序错误。
- 硬件组态不完整。

排查：

```text
切 RUN → 看诊断缓冲区 → 编译全部重建 → 确认 OB1 存在 → 重新下载
```

### 触发没反应

常见原因：

- OB1 没有调用目标 FB/FC。
- 输入写到了错误变量。
- 上升沿条件没有从 FALSE 变 TRUE。
- 复位信号一直为 TRUE。
- 互锁或报警未清除。

排查：

```text
监控 OB1 调用 → 监控 FB 输入 → Sim_Reset 清零 → Trigger FALSE 再 TRUE → 看状态机和报警
```

推荐仿真表字段：

```text
Sim_Reset
Sim_Trigger
In_Start / In_Stop
Stat_State
Out_Run / Out_Done / Out_Alm
Alm_* / Sts_*
```

## HMI / WinCC 排查

### HMI 读不到变量

常见原因：

- HMI 连接指向错误 PLC。
- HMI 变量路径错误。
- PLC DB 没有下载最新版本。
- 优化块访问与 HMI 访问方式不兼容。
- HMI 运行系统未重新下载。

排查：

```text
检查 HMI 连接 → 重新浏览 PLC 变量 → 重新生成 HMI 变量 → 下载 PLC → 下载 HMI
```

### 按钮无效

常见原因：

- 按钮事件没有绑定变量。
- 变量写权限或连接错误。
- PLC 逻辑马上复位该命令。
- 使用了脉冲按钮但 PLC 需要保持信号。

排查：

```text
在线监控 HMI 变量 → 按按钮看 PLC 变量是否变化 → 检查命令复位逻辑 → 改为置位/复位或脉冲
```

### 报警不显示

常见原因：

- 报警位没有进入 HMI 报警表。
- 报警类别/触发位配置错误。
- HMI 没下载最新报警组态。
- PLC 报警位只闪一下，没有锁存。

排查：

```text
监控 PLC 报警位 → 检查 HMI 报警变量 → 检查报警类别 → 下载 HMI → 测试锁存/复位
```

## 通信排查

### PROFINET

常见问题：

- IP 不同网段。
- 设备名未分配或不一致。
- GSDML 文件不匹配。
- IO 地址和程序地址不一致。
- 模块型号、槽位、子模块不一致。
- 网线、交换机、端口、拓扑问题。

排查顺序：

```text
可访问设备 → IP/设备名 → 在线诊断 → 模块状态 → IO 地址 → 程序引用
```

### PROFIBUS

常见问题：

- 站地址冲突。
- 波特率不一致。
- 终端电阻错误。
- GSD 文件不匹配。
- DP 主站/从站组态错误。
- 线缆屏蔽或接头问题。

排查顺序：

```text
站地址 → 波特率 → 终端电阻 → 在线诊断 → GSD/模块 → IO 地址
```

### S7 通信 / HMI 通信

常见问题：

- 连接资源不足。
- 访问保护等级限制。
- 伙伴 IP 或 TSAP 错误。
- 防火墙或 NAT 阻断。
- DB 访问方式不兼容。

排查：

```text
检查连接参数 → 检查访问保护 → Ping/可访问设备 → 监控连接状态 → 检查 DB 访问属性
```
