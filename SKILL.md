# 西门子博图 TIA Portal 技能

适用于 TIA Portal / STEP 7 V16/V17 自动化工程任务；V18+ 项目需要注意界面、库、CPU 固件和编译诊断差异。
重点覆盖 S7-1200、S7-1500，也支持 S7-300/S7-400 旧项目维护。

核心能力：

- 生成、审查、修改 PLC 程序块：OB / FB / FC / DB / UDT。
- 处理 LAD / FBD / SCL / STL / GRAPH 语言问题。
- 处理 SCL 源文件导入、编码、变量声明、多实例调用。
- 指导编译、下载、PLCSIM 仿真、在线监控、HMI 与通信排查。
- 输出面向现场可执行的步骤，而不是只讲概念。

## 维护元信息

- 适用版本：默认 TIA Portal V16/V17；V18+ 需按实际 CPU 固件、库版本、授权和诊断信息复核差异。
- 更新日期：2026-06-02。
- 已知限制：不能替代现场安全评估、企业变更流程、功能安全验证或 Siemens 官方兼容性矩阵；真实 PLC 操作必须由现场授权人员确认。

当前默认背景：

- 常用版本：TIA Portal V16/V17。
- 常用 CPU：S7-1200 / S7-1500。
- 常用仿真：PLCSIM。
- 本地文件路径：如有用户专属路径约定，先查工作区 `TOOLS.md`，不要写死在通用技能中。
- 生成 `.scl` 文件必须使用 **UTF-8 BOM**。
- FB 多实例：接口赋值后必须显式调用，再读取输出。

## 快速开始

收到博图相关任务后，先按这个顺序判断：

```text
用户需求 → 判断任务类型 → 定位资料/文件 → 编写或排查 → 给出导入/编译/下载/仿真步骤 → 提醒风险
```

如果用户给 `.scl` 文件或要求生成代码：

```text
确认块类型 → 确认接口变量 → 写 SCL → 检查 # 前缀 → 检查多实例调用 → UTF-8 BOM 保存 → 给导入和编译步骤
```

如果用户说“报错、编译不过、下载失败、仿真不动、HMI 不显示”：

```text
复述现象 → 要关键报错原文/截图信息 → 按编译/下载/仿真/HMI/通信分类 → 给最短排查链
```

高风险动作先停一下：

```text
真实 PLC 下载 / 强制变量 / 改硬件组态 / 覆盖 DB / 影响现场动作 → 先完成真实 PLC 操作安全检查清单，确认备份、停机/安全状态、授权
```

## 工作流一览

### 1. 生成新 SCL 代码

```text
确认需求 → 定义接口 → 选择 FB/FC/DB/UDT → 写完整变量声明 → 写逻辑 → 保存 UTF-8 BOM → 输出导入步骤
```

必须检查：

- 块类型：有状态设备优先 FB，纯计算优先 FC。
- 接口变量：输入、输出、InOut、静态、临时。
- `#` 前缀、状态机复位、多实例调用是否正确。

常用写文件命令：

```bash
python3 << 'PY'
from pathlib import Path
Path('Block.scl').write_text(content, encoding='utf-8-sig')
PY
```

检查 BOM：`xxd -l 3 Block.scl`，期望 `ef bb bf`。

### 2. 修改已有代码

```text
先读取原文件 → 找到目标块/变量 → 最小范围修改 → 保持编码 UTF-8 BOM → 说明改动点
```

不要盲改：

- 不要猜变量名。
- 不要删用户现有注释和结构。
- 不要重排整个文件，除非用户要求重构。
- 修改后检查 `IF/END_IF`、`CASE/END_CASE`、`FOR/END_FOR`。

### 3. 编译

TIA 中常用路径：

```text
项目树 → 选中 PLC 或程序块 → 编译 → 软件（全部重建）
```

排查优先级：

1. 变量未声明或作用域错误。
2. 局部变量缺少 `#` 前缀。
3. 数据类型不匹配。
4. FB 调用缺少实例 DB 或多实例声明。
5. SCL 结构未闭合、分号缺失、数组越界。
6. 源文件编码不是 UTF-8 BOM。

### 4. 下载

```text
设置 PG/PC 接口 → 搜索可访问设备 → 匹配 CPU → 下载硬件和软件 → CPU 切 RUN
```

下载前确认 CPU 型号、固件、IP、PROFINET 设备名、硬件组态一致。
真实 PLC 必须先确认备份、停机/安全状态和现场授权。

### 5. PLCSIM 仿真

```text
启动 PLCSIM → 下载硬件和软件 → CPU RUN → 打开监控表 → 写输入 → 看输出和状态
```

用户常用节奏：

```text
Sim_Reset 清零 → 设置参数 → Sim_Trigger FALSE → TRUE 触发 → 观察 Out_* 输出
```

仿真覆盖：复位、启动、停止、急停、正常完成、超时、互锁、异常输入、报警锁存和复位。

### 6. 在线监控与诊断

```text
CPU 状态 → 诊断缓冲区 → 硬件状态 → OB1 是否调用 → FB/FC 内部变量 → 输出和报警
```

常用工具：在线监控、监控表、强制表（危险）、交叉引用、在线诊断、Trace。

## 关键铁律

### 1. 现场安全第一

涉及真实 PLC 或现场设备时，先执行下方安全检查清单；不要直接建议“立刻下载/强制/覆盖”。

### 2. SCL 代码铁律

- 局部变量、接口变量、静态变量、多实例在 SCL 中通常用 `#` 访问。
- `.scl` 文件必须保存为 **UTF-8 BOM**。
- FB 多实例必须“先给接口赋值 → 再调用实例 → 再读取输出”。
- 状态机复位必须具备最高优先级，复位分支用 `ELSE` 包裹主 `CASE`。
- 详细规则、示例和模板见 `scl-rules.md`。

### 3. STL / GRAPH 边界

- STL 主要用于 S7-300/400 旧项目；S7-1200/S7-1500 通常不支持 STL 新建程序块，S7-1500 新项目应使用 LAD/FBD/SCL/GRAPH。
- TIA Portal 顺控图通常叫 GRAPH / S7-GRAPH，不要和 PCS7 的 SFC 混用。

## 安全提醒

以下情况不要直接给“立刻下载/强制/覆盖”的建议：

- 真实产线 PLC 下载。
- 强制输入输出或内部变量。
- 修改硬件组态、网络地址、PROFINET 设备名。
- 覆盖 DB 初始值、配方、累计量、校准参数。
- 可能导致电机、阀门、气缸、加热、泵动作的逻辑。

### 真实 PLC 操作强制安全检查清单

执行真实 PLC 下载、在线修改、强制变量、覆盖 DB 或可能影响现场动作的步骤前，必须逐项确认：

- 当前对象是真实 PLC 还是 PLCSIM；如果是真实 PLC，按现场变更流程处理。
- 已上传现场 PLC 程序，并归档当前项目和在线备份。
- 已确认 DB 当前值、配方、累计量、校准参数、保持值不会被下载或初始化覆盖。
- 已确认设备处于手动、停机或其它经现场认可的安全状态。
- 已确认是否涉及 F-CPU、安全程序、急停、安全门、光栅、双手按钮、安全扭矩关闭（STO）等功能安全内容。
- 已获得现场负责人或授权人员许可。
- 如果使用强制变量，操作结束后必须取消强制，并复查 force table / 强制表，确认无残留强制项。

### 安全联锁硬边界

- 不帮助绕过、屏蔽或削弱急停、安全门、光栅、双手按钮、安全扭矩关闭（STO）等安全功能。
- 涉及 F-CPU、安全程序或安全功能修改时，必须由具备资质的现场人员按企业流程评审、下载、验证和记录。

建议话术：`这一步会影响现场设备。先确认对象是真实 PLC 还是 PLCSIM；若是真实 PLC，请先上传并归档现场程序，确认 DB 当前值不会被覆盖，设备处于手动/停机/安全状态，并获得现场负责人授权后再操作。强制变量后必须取消强制并复查强制表。`
