# TIA Portal 编程语言说明

本文件用于快速判断 LAD / FBD / SCL / STL / GRAPH 的适用场景、常见写法和注意事项。

## 语言选择原则

| 语言 | 适合 | 不适合 | 建议 |
|---|---|---|---|
| LAD | 启停保持、互锁、继电器逻辑 | 复杂算法、数组循环 | 电气维护友好，现场监控直观 |
| FBD | 信号流、模拟量、功能块连接 | 大量文本算法、复杂状态机 | 适合图形化组合逻辑 |
| SCL | 算法、数组、结构体、状态机、代码生成 | 纯电气继电器图纸复刻 | 新项目文本逻辑首选 |
| STL | S7-300/400 旧项目维护 | S7-1200/S7-1500 新建程序块 | 只读懂和迁移；S7-1500 新项目使用 LAD/FBD/SCL/GRAPH |
| GRAPH | 顺序动作、步进流程、批处理 | 单纯组合逻辑、短小算法 | 步/转换/动作要定义清楚 |

## LAD（梯形图）

中文常称：梯形图。

### 适用场景

- 电机启停保持。
- 阀门开关互锁。
- 急停、安全许可、运行允许。
- 简单报警锁存。
- 电气人员需要在线看触点通断的逻辑。

### 常见结构

```text
启动条件 + 停止条件 + 自保持 → 运行输出
互锁条件 + 命令 → 动作输出
故障条件 → 报警锁存 → 复位清除
```

### 优点

- 接近继电器电路图。
- 在线监控直观。
- 现场电工容易理解。

### 注意事项

- 复杂嵌套网络会很难读。
- 数组、循环、字符串、报文处理不适合 LAD。
- 触点串并联过多时，建议拆成中间变量。
- 与 SCL 混用时，要统一变量命名和注释。

## FBD（功能块图）

中文常称：功能块图。

### 适用场景

- 模拟量限幅、滤波、缩放。
- 多个功能块信号连接。
- 逻辑门组合：AND / OR / NOT / 选择器。
- PID、定时器、计数器等块调用可视化连接。

### 常见结构

```text
输入信号 → 转换/滤波/限幅 → 功能块 → 输出状态
```

### 优点

- 信号流清楚。
- 功能块连接关系直观。
- 适合过程控制和模拟量处理。

### 注意事项

- 复杂状态机可读性差。
- 线太多时要分网络、加中间变量。
- 多块调用顺序要注意扫描周期中的执行顺序。

## SCL（结构化控制语言）

中文常称：结构化控制语言。

### 适用场景

- 状态机。
- 批量设备逻辑。
- 数组循环。
- UDT / Struct 数据处理。
- 配方、报文、算法、计算。
- 需要生成或版本管理的代码。

### 基本语法

```scl
IF #In_Start AND NOT #In_Stop THEN
    #Out_Run := TRUE;
ELSE
    #Out_Run := FALSE;
END_IF;
```

```scl
CASE #Stat_State OF
    0:
        IF #In_Start THEN
            #Stat_State := 10;
        END_IF;
    10:
        IF #In_Done THEN
            #Stat_State := 100;
        END_IF;
    ELSE
        #Stat_State := 0;
END_CASE;
```

```scl
FOR #Temp_i := 0 TO 9 DO
    #Out_Array[#Temp_i] := #In_Array[#Temp_i] * 2.0;
END_FOR;
```

### 必须注意

- 局部/接口变量加 `#`。
- 语句通常用 `;` 结束。
- `IF` 必须配 `END_IF`。
- `CASE` 必须配 `END_CASE`。
- `FOR` 必须配 `END_FOR`。
- 数组下标不要越界。
- `.scl` 源文件保存为 UTF-8 BOM。

## STL（语句表）

中文常称：语句表。

### 支持范围

- STL 主要用于 S7-300/400 旧项目；S7-1200/S7-1500 通常不支持 STL 新建程序块，S7-1500 新项目应使用 LAD/FBD/SCL/GRAPH。

### 适用场景

- 旧 STEP 7 项目维护。
- 阅读历史低层逻辑。
- 将存量 STL 迁移到 LAD / FBD / SCL。

### 注意事项

- 不要把 STL 当作新项目默认语言。
- 需要确认 CPU 系列和 TIA 版本支持情况。
- 间接寻址、累加器、状态字逻辑要谨慎解释。
- 迁移时优先保留行为，再优化结构。

## GRAPH（S7-GRAPH 顺控图）

TIA Portal 中的顺序控制一般叫 GRAPH 或 S7-GRAPH。
不要把它和 PCS7 的 SFC 混用。

### 适用场景

- 步进流程。
- 批处理动作序列。
- 设备启动/停止流程。
- 气缸、阀组、输送线顺序动作。
- 有明确步骤、转换条件、动作输出的控制。

### 基本组成

```text
步 Step → 动作 Action
转换 Transition → 下一步条件
分支 Branch → 选择或并行流程
复位 Reset → 回初始步或安全步
```

### 设计要点

- 每一步只做清晰动作。
- 每个转换条件可监控、可解释。
- 异常、超时、急停要有处理路径。
- 要定义上电初始步和复位策略。
- 输出动作要避免多个步同时冲突。

### 注意事项

- GRAPH 支持情况与 TIA 版本、CPU、授权有关。
- 小型简单逻辑不一定需要 GRAPH。
- 复杂流程建议配合状态、报警、超时监控。
